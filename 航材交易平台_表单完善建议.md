# AeroLink 航材智能销售系统 — 各模块表单完善建议

> 基于《测试软件补充要素》梳理结果，结合 CAAC AC-120-FS-058、FAA AC 20-154、ATA Spec 106/2000、ILS 行业标准及 Incoterms® 2020 等国际国内航材贸易规范编制。

---

## 二、分模块详细完善建议

### 2.1 询价管理 (RFQ Management)

#### 2.1.1 新增字段

| 字段名 | 类型 | 必填 | 说明 | 行业标准依据 |
|--------|------|------|------|--------------|
| `uom` | String | 是 | 计量单位：EA(个)、PCS(件)、PC(片)、SET(套)、KG、LB、FT、M | ILS / 航材贸易惯例 |
| `conditionCode` | String | 是 | 状态代码：**NE**(全新)、**NS**(新件余料)、**OH**(大修件)、**SV**(可用件)、**AR**(拆机件)、**FN**(工厂新件)、**RP**(修理件) | ILS Condition Codes / FAA AC 20-154 |
| `description` | String | 否 | 航材完整描述（可自动从 IPC 数据带出） | ATA Spec 100 / IPC |
| `leadTimeDays` | Int | 否 | 客户期望交货天数 | 采购惯例 |
| `responseDeadline` | DateTime | 否 | 报价截止日期（自动计算：创建日 + N 工作日） | ILS RFQ 标准流程 |
| `ataChapter` | String | 否 | ATA 章节号（如 32-10-01），用于分类和检索 | ATA Spec 100 |
| `serialNumber` | String | 否 | 序号（时寿件/高价值件必填） | CAAC AC-120-FS-058 6.1 |
| `batchNumber` | String | 否 | 批次号（标准件/原材料适用） | CAAC AC-120-FS-058 6.3 |
| `aircraftModel` | String | 否 | 具体机型（如 B737-800、A320CEO），与 `aircraftType` 区分 | 适航性要求 |
| `certificateRequired` | Boolean | 是 | 是否需要适航证书 | CAAC AC-120-FS-058 |
| `certificateType` | String | 否 | 适航证类型：AAC-038 / FAA-8130-3 / EASA-Form-1 / COC | CAAC / FAA / EASA |
| `targetPriceCurrency` | String | 否 | 目标价格货币：USD / EUR / CNY，默认 USD | 国际贸易惯例 |
| `alternatePartNumbers` | String[] | 否 | 可互换件号列表（从 IPCData 关联） | IPC / PMA 数据 |
| `urgencyJustification` | String | 否 | AOG/紧急询价的理由说明 | 内部审批需要 |

#### 2.1.2 字段改造建议
- `partNumber`：增加 **IPC 自动校验**，输入时联想匹配，带出 `description`、`ataChapter`、`aircraftTypes`、`alternateParts`
- `aircraftType`：建议改为多选或级联选择（先选 Manufacturer → 再选 Model → 再选 MSN 范围）
- `urgency`：AOG 类型需强制填写 `urgencyJustification` 并触发审批流

| `shipForAddress` | String | 否 | 开票/账单地址 | 财务结算 |
| `shippingContactName` | String | 否 | 收货联系人 | 物流对接 |
| `shippingContactPhone` | String | 否 | 收货联系人电话 | 物流对接 |

| `vatNumber` | String | 否 | 增值税号 / 统一社会信用代码 | 税务合规 |

| `aocNumber` | String | 否 | 航空运营人合格证编号 | CAAC 合规 |
| `creditRating` | String | 否 | 信用评级：A / B / C / D | 风控管理 |
| `paymentMethod` | String | 否 | 支付方式：Wire / LC / Credit Card / OA | 财务条款 |

#### 2.2.2 字段改造建议
- `address` → 拆分为：`registeredAddress`（注册地址）、`shipToAddress`（收货地址）、`shipForAddress`（账单地址）
- `contactName` / `email` / `phone` → 保留为**主联系人**，新增 `contacts` 子表支持多联系人
- `paymentTerms`：建议标准化为枚举值（Net 30 / Net 60 / Net 90 / Prepaid / 50% Deposit）

### 2.3 报价单 (Quotations)

#### 2.3.1 新增字段

| 字段名 | 类型 | 必填 | 说明 | 行业标准依据 |
|--------|------|------|------|--------------|
| `saleType` | String | 是 | 销售类型：**Sale**(直接销售)、**Exchange**(交换)、**Loan**(租借)、**Consign**(寄售)、**Repair**(送修) | 航材贸易模式 |
| `shipToId` | String | 否 | 收货地址 ID（关联客户子地址） | 物流交付 |
| `shipForId` | String | 否 | 开票地址 ID（关联客户子地址） | 财务结算 |
| `incoterm` | String | 是 | 国际贸易术语：EXW / FCA / CPT / CIP / DAP / DPU / DDP / FAS / FOB / CFR / CIF | Incoterms® 2020 |
| `incotermLocation` | String | 否 | Incoterm 指定地点（如 DDP 北京、FOB 上海） | Incoterms® 2020 |
| `leadTimeDays` | Int | 是 | 承诺交货天数（从订单确认起算） | 报价核心要素 |
| `leadTimeBasis` | String | 否 | 交货期基准：Stock(现货) / AOG(紧急调拨) / PO to OEM(原厂订货) | 交付说明 |
| `ccRecipients` | String[] | 否 | 抄送邮箱列表 | 邮件分发 |
| `commonNote` | String | 否 | 通用备注/特殊条款（支持模板变量） | 合同条款 |
| `moq` | Int | 否 | 最小起订量 (Minimum Order Quantity) | 贸易惯例 |
| `mpq` | Int | 否 | 最小包装量 (Minimum Pack Quantity) | 贸易惯例 |
| `priceBasis` | String | 否 | 价格基准：Ex-Works / Delivered / Installed | 定价基准 |
| `taxIncluded` | Boolean | 是 | 是否含税（默认 true，出口可 false） | 税务合规 |
| `taxRate` | Float | 否 | 税率（如 13% / 0%） | 税务合规 |
| `warrantyDays` | Int | 否 | 质保天数（默认 90 天，NE 件可 365 天） | CAAC 质保要求 |
| `warrantyTerms` | String | 否 | 质保条款描述 | 合同条款 |
| `validityDays` | Int | 是 | 报价有效期天数（建议默认 7 天，AOG 1 天） | 贸易惯例 |
| `validityDeadline` | DateTime | 否 | 报价有效期截止日期（自动计算） | 系统生成 |
| `packagingRequirement` | String | 否 | 包装要求：ATA-300 / Anti-Static / Moisture-Barrier / OEM-Box | ATA-300 标准 |
| `shippingMethod` | String | 否 | 运输方式：Courier / Air Freight / Sea Freight / Road | 物流方式 |
| `inspectionStandard` | String | 否 | 检验标准：AQL 1.0 / 100% Inspection / Sampling Plan | 质检标准 |
| `inspectionReportIncluded` | Boolean | 否 | 是否附带检验报告 | 质量文件 |
| `certificateOfConformance` | Boolean | 否 | 是否提供 COC（合格证） | CAAC 6.3 |
| `countryOfOrigin` | String | 否 | 原产国（如 US / CN / DE） | 报关要素 |
| `hsCode` | String | 否 | 海关编码（如 8803.30） | 报关要素 |
| `eccn` | String | 否 | 出口管制分类编码（如 9A991） | 出口管制 |
| `dualUse` | Boolean | 否 | 是否军民两用物项 | 出口管制 |
| `eSignature` | JSON | 否 | 电子签名数据（签名图片、时间戳、签名人 ID） | 合同法律效力 |
| `eSignatureStatus` | String | 否 | 签名状态：Unsigned / Signed / Rejected | 合同流程 |

#### 2.3.2 字段改造建议
- `deliveryTerms` → 建议改为 `incoterm` + `incotermLocation` 组合，符合国际贸易标准
- `paymentTerms` → 标准化枚举 + 自定义文本补充
- `certificateFiles` → 扩展为 `attachments` 数组，支持多类型文件（COC、8130-3、检验报告、照片等）
- `customerConfirmationNote` → 保留，但增加 `customerSignature` 字段支持客户电子确认

### 2.7 合同/文档模板 (Document Templates)

#### 2.7.1 新增字段与功能

| 功能/字段 | 说明 | 优先级 |
|-----------|------|--------|

## 五、实施优先级建议

### P0（立即实施）— 业务阻断级
1. **RFQ 增加 UOM、Condition Code、Response Deadline** — 缺少单位导致报价歧义，缺少状态代码无法区分新件/二手件
2. **报价单增加 Incoterms、Lead Time、Warranty Days、Tax Included** — 国际贸易核心条款缺失
3. **客户增加 Ship To / Ship For 地址分离** — 当前单一地址无法满足物流和财务分离需求
4. **库存增加 Serial Number、Condition Code、Certificate Type/Number** — 适航追溯核心属性缺失

### P1（短期实施）— 效率提升级
5. **RFQ 增加 ATA Chapter、Aircraft Model、Certificate Type** — 提升寻源匹配精度
6. **报价单增加 Sale Type、MOQ/MPQ、Packaging、Shipping Method** — 完善商务条款
7. **客户增加 Buyer Type、多联系人、VAT、IATA/ICAO** — 客户画像完善
8. **库存增加 Life Limited、Hours/Cycles、Shelf Life、AD/SB Status** — 时寿件管理必备
9. **合同电子签名** — 法律效力与无纸化

### P2（中期实施）— 合规与竞争力级
10. **库存增加 Traceability Docs、Previous Operator、Removal Info、NIS** — 二手件完整追溯（ATA 106）
11. **供应商增加 145 Certificate、OEM Authorization、Audit Date、CAGE Code** — 供应商资质合规
12. **报价单增加 HS Code、ECCN、Dual Use、Country of Origin** — 进出口合规
13. **订单增加 Customs Clearance、Import Duty、Total Land Cost** — 到岸成本核算

### P3（长期规划）— 智能化级
14. **IPC 数据自动关联** — 件号输入自动带出 Description、ATA、互换件号
15. **证书 OCR 自动识别** — 上传 8130-3 / AAC-038 自动提取序号、件号、证书编号
16. **时寿件预警** — 基于 Shelf Life / Hours / Cycles 的自动预警
17. **AOG 智能审批流** — AOG 询价自动触发快速审批通道
