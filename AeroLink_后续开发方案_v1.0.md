### 1.1 市场规模

| 2025-2034 CAGR | **4.7%** | GM Insights |

| **自动化工作流编排** | 成长期 | 2026-2028 | **高优先级** — 电子工卡、审批流、RFQ自动响应 |
| **统一数据生态/仪表板** | 成长期 | 2026-2028 | **高优先级** — 集成多源数据，统一KPI |

- **FAA/EASA/ICAO** 要求维修、零部件、技术员认证的数字化跟踪
- **CAAC AC-120-FS-058 R3** 对合格航材的追溯性要求日益严格
- 适航数据自动归集成为审定硬性要求（参考ITL案例）
- 数据安全和网络安全合规成为云化前提

4. **集成层**：缺少API开放、ERP对接、外部数据接入
5. **证书层**：缺少电子证书签发、链上存证

## 三、开发方案总体设计

### 3.1 战略定位

> **AeroLink 定位**：面向中小型航材贸易商和MRO企业的**智能化航材交易与供应链协同平台**，以"降本、增效、合规"为核心价值主张。

### 3.2 开发原则

1. **价值优先**：每个阶段必须有明确的商业价值和用户痛点解决
2. **渐进智能**：从"规则自动化"逐步过渡到"AI智能化"，避免过度工程
3. **合规嵌入**：将CAAC/FAA/EASA合规要求内建到流程中，而非事后补丁
4. **开放集成**：预留API和Webhook，为后续ERP对接和生态扩展做准备
5. **数据驱动**：建立数据埋点和分析能力，为AI模型训练积累数据资产

### 3.3 技术架构演进方向

```
当前架构（已完成）
├── 前端: React 19 + Vite + Tailwind + shadcn/ui + Zustand
├── 后端: Node.js + Express + Prisma + SQLite + Socket.IO
└── 数据: 本地SQLite，种子数据

Phase 1（近期）: 功能补全 + 数据升级
├── 数据库: SQLite → PostgreSQL（生产环境）
├── 缓存: 引入Redis（会话、热点数据）
├── 搜索: 引入Elasticsearch（件号模糊搜索、全文检索）
└── 文件: 引入对象存储（证书PDF、合同扫描件）

Phase 2（中期）: 智能化 + 自动化
├── AI层: 引入LLM API（需求预测、智能报价、文档解析）
├── 工作流: 引入工作流引擎（审批流、电子工卡、状态机）
├── 消息: 引入消息队列（异步处理、事件驱动）
└── 集成: 开放REST API + Webhook

Phase 3（远期）: 平台化 + 生态
├── 多租户: SaaS化架构
├── 区块链: 证书链上存证（Hyperledger/Fabric）
├── 大数据: 行业价格指数、FMV计算
└── 移动端: PWA/小程序
```

## 四、分阶段开发路线图

### Phase 1：功能补全期（预计 6-8 周）

**目标**：补齐基础交易平台的必备功能，达到"可用"标准。

#### 1.1 电子证书管理（Certificate Manager）

**行业依据**：
- IATA SmartHub 2022路线图明确将"SmartDocs — 数字证书签发"列为核心模块
- CAAC AC-120-FS-058 R3 要求合格航材必须具备可追溯的证书链
- FAA AC 20-154 对AAC-038、FAA 8130-3、EASA Form 1的数字化管理提出明确要求

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 证书模板管理 | P0 | 支持AAC-038、FAA 8130-3、EASA Form 1、COC模板 |
| 证书签发 | P0 | 与Inventory/Order关联，一键生成PDF证书 |
| 证书验证 | P1 | 二维码/链接验证证书真伪 |
| 证书链追溯 | P1 | 记录证书流转历史（签发→转让→核销） |
| 到期预警 | P1 | 证书有效期倒计时，自动提醒 |

**技术方案**：
- 后端：新增 `Certificate` 模型，关联 Inventory/Order/Supplier
- 前端：证书签发弹窗、证书列表页、验证页面
- 文件存储：证书PDF生成（使用PDF库如pdf-lib或puppeteer）

#### 1.2 工作流引擎（Workflow Engine）

**行业依据**：
- ITL智慧MRO以"电子工卡为主线"覆盖端到端维修业务流程
- IATA SmartHub强调"自动化MRO工作流编排"
- 东航技术强调"人-机-现场-后台感知闭环"

**功能设计**：

| 工作流 | 触发条件 | 审批节点 | 自动化动作 |
|--------|---------|---------|-----------|
| **RFQ审批流** | RFQ创建 | 销售经理→总监 | 超期自动升级、AOG自动跳过 |
| **报价审批流** | 报价提交 | 销售经理→财务 | 折扣超阈值自动升级 |
| **订单审批流** | 订单确认 | 销售→财务→物流 | 信用额度自动校验 |
| **AOG紧急流** | AOG标记 | 并行审批（2人） | 1小时未响应自动升级 |
| **证书签发流** | 出库完成 | 质量经理 | 自动关联库存记录 |
| **供应商准入流** | 新供应商注册 | 采购→质量→合规 | 资质自动校验 |

**技术方案**：
- 引入状态机库（如xstate或自研轻量状态机）
- 新增 `WorkflowInstance` / `WorkflowStep` / `WorkflowAction` 模型
- 前端：审批待办列表、审批历史时间线、审批操作按钮
- 通知：Socket.IO实时推送 + 邮件通知（预留）

#### 1.3 数据升级与性能优化

| 任务 | 优先级 | 说明 |
|------|--------|------|
| SQLite → PostgreSQL | P0 | 生产环境迁移，支持并发和大数据量 |
| 件号搜索优化 | P0 | 当前前端过滤，需后端模糊搜索+分页 |
| 图片/文件存储 | P1 | 证书PDF、合同扫描件、产品图片 |
| 数据导入导出 | P1 | Excel批量导入库存/客户/供应商 |
| 操作日志审计 | P1 | 记录所有关键操作（谁、何时、改了什么） |

### Phase 2：智能化升级期（预计 8-12 周）

**目标**：引入AI和自动化能力，实现"降本增效"的核心价值。

#### 2.1 AI智能报价（AI Quotation Assistant）

**行业依据**：
- 行业数据显示**50-80%的RFQ可以通过自动报价响应**
- ePlaneAI以AI驱动检索和报价为核心卖点
- Revima平台承诺"24小时内获取报价"

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **历史价格推荐** | P0 | 基于相同件号的历史报价/成交价，自动推荐价格区间 |
| **竞品价格监控** | P1 | 抓取ILS/PartsBase公开价格（合规前提下），给出市场参考 |
| **智能折扣建议** | P1 | 基于客户等级、采购量、账期，AI建议折扣率 |
| **自动报价生成** | P2 | 对于标准件/常用件，RFQ到达后自动生成为草稿报价 |
| **报价胜率预测** | P2 | 基于历史数据，预测当前报价的赢单概率 |

**技术方案**：
- 规则引擎（近期）：基于件号+客户+数量的历史均价计算
- ML模型（中期）：训练价格预测模型（需积累至少1000+交易记录）
- LLM辅助（远期）：自然语言解析RFQ邮件/文档，自动提取需求

#### 2.2 需求预测与库存优化（Demand Forecasting）

**行业依据**：
- 预测性维护是2027年全面采用的关键技术
- IATA SmartHub的Evaluator模块提供市场数据分析
- 东航技术将"故障预测、维修排程与库存优化"列为高优先级

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **消耗趋势分析** | P0 | 按件号/客户/机型统计历史消耗量，生成趋势图 |
| **安全库存建议** | P1 | 基于消耗趋势和交货周期，计算建议安全库存量 |
| **补货提醒** | P1 | 库存低于安全线时自动提醒采购 |
| **季节性预测** | P2 | 识别季节性波动（如春运前航材需求高峰） |
| **AOG需求预测** | P2 | 基于机型机队规模和故障率，预测高需求件号 |

**技术方案**：
- 使用简单统计模型（移动平均、指数平滑）作为起点
- 数据积累后引入Prophet或LSTM等时序预测模型
- 前端：Dashboard新增"库存健康度"和"补货建议"卡片

#### 2.3 拍卖与反向拍卖（Auction Module）

**行业依据**：
- IATA SmartHub的Auctioneer模块是核心差异化功能
- 反向拍卖可降低采购成本10-15%（IATA数据）
- 销售拍卖可加速呆滞库存变现

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **销售拍卖** | P1 | 设置起拍价、保留价、时间，多买家竞价 |
| **反向拍卖（采购）** | P1 | 发布采购需求，多家供应商竞价，自动选最低价 |
| **密封投标** | P2 | 投标价保密，截止后统一开标 |
| **自动延时** | P2 | 截止前5分钟有新出价，自动延长5分钟 |

**技术方案**：
- 新增 `Auction` / `AuctionBid` 模型
- Socket.IO实时推送竞价更新
- 前端：拍卖大厅页面、出价弹窗、倒计时组件

#### 2.4 寄售管理（Consignment Module）

**行业依据**：
- IATA SmartHub Consignment模块：消除电子表格和定期状态会议
- 寄售是航材行业常见模式（供应商库存放在买方仓库，使用时结算）

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **寄售协议管理** | P1 | 记录寄售期限、结算方式、最低库存 |
| **寄售库存跟踪** | P1 | 区分自有库存和寄售库存 |
| **消耗结算** | P1 | 按实际消耗自动生成结算单 |
| **寄售预警** | P2 | 寄售库存不足/超期提醒 |

### Phase 3：平台化与生态期（预计 12-16 周）

**目标**：从"工具"升级为"平台"，构建行业数据生态。

#### 3.1 公正市场价值（FMV）引擎

**行业依据**：
- IATA SmartHub的核心价值：150万+件号的FMV数据
- FMV算法采用分层统计方法，考虑件号、制造商、条件、日期、互用性
- 可帮助用户降低10-15%材料成本

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **价格数据采集** | P0 | 从平台内交易记录、公开数据源采集价格 |
| **FMV计算引擎** | P1 | 分层算法：同件号近期交易→相似件号→互用件→机型均价 |
| **价格趋势图** | P1 | 件号价格历史走势 |
| **估值报告** | P2 | 输入库存清单，一键生成总估值报告 |
| **行业基准** | P2 | 与同行业企业匿名对标 |

**技术方案**：
- 数据层：建立价格数据仓库（Time-Series DB或PostgreSQL分区表）
- 算法层：从简单统计开始，逐步引入ML模型
- 前端：件号详情页增加"市场参考价"区域

#### 3.2 API开放平台与ERP集成

**行业依据**：
- IATA SmartHub提供完整API，可集成到任何ERP/库存系统
- ILS/PartsBase均提供API接口
- 企业客户的核心诉求："不要让我改ERP"

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **REST API v1** | P0 | 核心资源（RFQ/报价/订单/库存）的CRUD API |
| **API Key管理** | P0 | 企业客户申请API Key，限流/权限控制 |
| **Webhook** | P1 | 事件触发外部系统回调（如订单状态变更） |
| **SAP/Oracle连接器** | P2 | 预置常见ERP系统的数据同步模板 |
| **Excel/CSV同步** | P1 | 最低成本的"集成"方式，定时导入导出 |

#### 3.3 电子证书链上存证（Blockchain Certificate）

**行业依据**：
- IATA SmartHub 2022路线图："Issuing new digital material certificates"
- 区块链在MRO追溯中的应用预计2028-2030年普及
- CAAC/FAA对证书真实性的要求日益严格

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **证书哈希上链** | P2 | 证书PDF的SHA-256哈希写入区块链（如蚂蚁链、Hyperledger） |
| **链上验证** | P2 | 扫描证书二维码，验证链上哈希匹配 |
| **流转记录上链** | P3 | 证书每次转让记录上链，形成不可篡改追溯链 |

#### 3.4 多语言与国际化

**行业依据**：
- 航材交易天然国际化（买家/卖家可能分布在不同国家）
- ILS/PartsBase/IATA均为全球平台

**功能设计**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **中英双语** | P1 | 界面、邮件、证书模板支持中英文切换 |
| **多币种** | P1 | 支持USD/EUR/CNY等，实时汇率转换 |
| **时区适配** | P2 | 报价截止、交货期按用户时区显示 |
| **多计量单位** | P2 | EA/KG/LB等自动转换 |
