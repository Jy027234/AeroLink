# P2-02/P2-03 Query 与 Feature 首批纵切

日期：2026-07-19（Asia/Shanghai）
状态：首批完成，整体收口进行中（RFQ、Quotation、Order、Inventory、Customer、Supplier 及 RFQ/Quotation 集成入口已迁移）

## 已交付

- 引入 TanStack Query 5，统一 `QueryClient` 默认 stale/cache、重试和 mutation 策略；生成客户端 mutation 适配器为写请求生成稳定的 `Idempotency-Key`，401 refresh 重试沿用同一 key。
- 建立 `src/features/<domain>/{api,queries,index}` 六个核心域及 `integrations` 公开入口；新适配层只调用 OpenAPI 生成客户端。
- `useApi.ts` 的六域列表/详情查询改由 Query 管理，保留返回形状兼容旧页面；RFQ、Quotation 创建/更新/状态/发送/撤回/接受、Order 创建/更新、Inventory 创建/更新、Customer 创建/更新和 Supplier 创建/更新均已接入生成客户端与精确失效。
- 订单详情弹窗已移除本地 `getById`/detail state 请求，改用 `useOrder(id, enabled)` feature query；弹窗关闭时不发请求，打开后共享订单 detail cache，详情 Query 首次失败立即显示错误（`retry: 0`），回退列表数据并支持显式 refetch 重试。
- 401 查询会尝试现有 refresh-cookie 流程；账号退出或用户 ID 变化时清空 Query cache，避免跨账号残留。
- `scripts/check-feature-boundaries.mjs` 与 `npm run features:check` 已接入 CI，禁止跨 feature 内部路径穿透。
