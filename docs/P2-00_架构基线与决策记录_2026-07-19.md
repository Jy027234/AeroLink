## 1. 基线结论

P2 从可运行的 P1 主链进入可维护性与运营阶段。当前 RFQ → 报价 → 审批 → 订单 → 库存 → 证书主链保持业务真相在服务端，核心写入已有事务、状态迁移、幂等、Outbox、权限和审计基础。

本包不改变业务运行行为，完成以下可复现清册和决策冻结：

- API 入口清册覆盖当前 `server/src/index.ts` 的 54 个 `app.use` HTTP mount，以及健康检查 `GET /api/health`；原始 P2-00 基线计数仍保留在表格中，新增入口均由路由漂移脚本追踪。
- 前端服务端状态清册确认 `src/api/client.ts`、`src/hooks/useApi.ts`、Zustand stores 和页面直连的当前责任，作为 P2-01～P2-03 迁移基线。
- 结构热点清册确认大型页面、路由、数据访问层、API 进程内定时器和通用文件入口。
- 冻结 6 项架构 ADR：OpenAPI 真相源、生成客户端、Query 缓存、模块化单体、数据库 Worker/租约、对象存储与观测。
- 修正 E2E UI 审计脚本的固定端口，使其使用 Playwright `baseURL`；该改动不改变业务行为，但使交接文档声明的隔离端口可复现。

## 6. ADR 决策记录

### ADR-001：OpenAPI 3.1 作为 HTTP 真相源

**决定**：在 `contracts/openapi/openapi.yaml` 使用 OpenAPI 3.1 文档作为所有已启用 HTTP 路由的契约真相源；按领域拆分引用文件，但生成和校验从一个稳定入口执行。

**取舍**：保留现有 Express 路由实现，契约先覆盖核心六域和特殊响应，再覆盖其余启用路由；不从运行时反射生成不稳定文档，也不为文档静默改变状态码、错误码、日期或金额表示。

**回滚**：P2-01 失败时保留旧路由/DTO，删除生成物和 CI 校验，不改数据库与业务 URL。

### ADR-002：生成型 TypeScript 客户端作为唯一新 HTTP 适配层

**决定**：P2-01 采用 OpenAPI 文档生成 TypeScript 类型和客户端，生成目录禁止手工编辑；新 feature 只依赖生成客户端封装，旧 `src/api/client.ts` 以按域兼容适配方式退出。

**取舍**：生成物可能增加构建步骤和类型校验时间，但能消除前后端 DTO/枚举漂移；客户端生成工具的 Node 22 兼容性、文档校验和包体积在 P2-01 开始前按当时官方文档复核并记录。

**回滚**：保留当前手写 client 的薄适配入口，关闭生成漂移门禁；不得保留两套新业务调用路径。

### ADR-003：TanStack Query 管理服务端状态

**决定**：P2-02 引入 TanStack Query v5，按域建立 query key factory，统一取消、重试、stale/cache、错误映射和 logout 清缓存；URL 继续承载可分享的列表状态。

**取舍**：Zustand 只保留 UI、草稿和会话状态；mutation 按场景选择精确更新、失效或可回滚乐观更新，不把 Query cache 当 URL 替代，也不在失败时返回 `null` 伪装成功。

**回滚**：每个域独立迁移，可在 feature 入口切回旧 hook；迁移提交不得删除旧实现，直到零引用和回归证据完成。

### ADR-004：后端保持模块化单体

**决定**：保持单仓库、单 API 部署单元，建立 `server/src/modules/<domain>` 的 routes/controller、schemas、service、policy、repository、events/tests 边界；旧 URL 由薄适配层保留。

**取舍**：跨模块写入只能通过明确用例/事件，避免路由串联多次 Prisma 写入；不引入微服务、Redis 或 Elasticsearch，不以目录完整性创建空文件。

**回滚**：每个业务域可回退到原 route adapter；数据库 migration 仅使用前向兼容策略，不修改已部署历史 migration。

### ADR-005：Worker 默认使用 PostgreSQL 领取/租约

**决定**：P2-05 复用 Transactional Outbox，以 PostgreSQL 原子领取、`workerId`、租约到期、心跳、指数退避、终态失败和受控重放实现独立 Worker；API 进程不再启动后台任务。

**取舍**：先不引入 Redis 队列，避免没有并发指标时增加基础设施；外部副作用以稳定幂等标识去重，滚动发布要求任务 schema 向后兼容。

**回滚**：保留旧处理函数作为明确的短期兼容入口，必要时暂停领取而不停止 API 业务写入；重放必须有权限、确认和审计。

### ADR-006：对象存储抽象与供应商中立观测

**决定**：P2-06 定义 `ObjectStorage` 接口，提供本地开发/测试适配器和 S3 兼容适配器；数据库只保存不可变对象元数据、SHA-256、对象引用和保留状态。P2-07 使用 Pino 结构化日志、request ID、指标及可关闭的供应商中立 trace/错误上报适配层。

**取舍**：先验证 MinIO 或等价隔离服务，不接真实云密钥；监控采集端不可用时业务必须降级继续，默认不记录 Cookie、Authorization、Token、密码、输入框内容和敏感查询参数。

**回滚**：历史 `/uploads` 源文件在对账和人工确认前不清理；对象迁移可断点续跑、核验后回退到旧引用；观测适配层可关闭而不影响业务请求。
