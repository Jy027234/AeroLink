## 8. 开发与安全约束

1. 本地仓库是开发位置，GitHub 是源码与版本基线；云端目录只用于部署/验证，不直接长期开发。
2. 只使用常规 Git 完成提交和推送；不提交任何环境机密、数据库或上传文件。
3. PostgreSQL 正式路径只使用 migration；每次 schema 变更须提供 migration、回填、对账、回滚说明和真实 PostgreSQL 验证。
4. 不恢复 localStorage bearer/refresh token，不为测试降低 Cookie 安全属性。
5. 可重试的外部写操作必须评估幂等键、唯一约束和异步副作用重复投递。
6. 没有真实数据时必须显示“无数据/样本不足”或关闭功能，不生成随机经营指标。
7. 未获重新授权前，不进行域名、DNS、TLS 或公网生产切流。
8. 发现非本任务改动时保留它们；不使用 destructive reset/checkout。
