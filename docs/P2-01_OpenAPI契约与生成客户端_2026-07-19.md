## 1. 本批目标与范围

本批在不改变现有 HTTP 行为的前提下建立 OpenAPI 3.1 真相源、路由漂移门禁、生成型 TypeScript 类型和兼容客户端。契约已覆盖认证/设备会话边界、RFQ → Quotation → Order → Inventory 主链、Customer/Supplier information 主数据的列表/创建/详情/更新/状态动作/历史/导出/PDF/库存对账，以及证书/文档、Webhook（含 Phase 2）、入站 Webhook、Outbox、用户只读、邮件读取/分类和邮箱账户管理的首批真实 DTO 边界。

明确未在本批完成的内容：外部供应商门户、域名/TLS、市场数据、AI 自主决策和云端采集留存仍按 P2 范围边界排除；已启用 HTTP 路由不再保留 baseline JSON envelope。

## 2. 交付物

- `contracts/openapi/openapi.json`：315 个已注册 HTTP operation 的 OpenAPI 3.1 真相源；54 个 mount 均可追溯到 `x-aerolink-source`。
- `src/api/generated/openapi.d.ts`：由 `openapi-typescript` 可重复生成的类型文件，禁止手工编辑。
- `src/api/generated/client.ts`：基于 `openapi-fetch` 的带 Cookie、内存 access token 和 `/api` origin 兼容客户端工厂；现有 `src/api/client.ts` 继续作为兼容层保留，P2-02/03 新迁移入口统一走生成客户端。
- `scripts/openapi/route-catalog.mjs`：从 `server/src/index.ts` 和路由文件发现 mount/operation。
- `scripts/openapi/generate-scaffold.mjs`：可审计的骨架和核心 DTO overlay 生成器；普通 `api:scaffold` 不覆盖已有真相源，重置必须显式使用 `api:scaffold:reset`。
- `scripts/openapi/check-route-coverage.mjs`：方法/路径一一对应、operationId、响应、来源和延期责任检查；严格模式会拒绝遗留 inventory operation。
- `scripts/openapi/check-generated.mjs`、`redocly.yaml` 和 `.github/workflows/ci.yml`：生成物漂移与 OpenAPI lint 的本地/CI 门禁。
- `src/api/openapi-contract.test.ts`、`src/api/generated/client.test.ts`：认证 Cookie、分页、幂等、上传/PDF/Webhook、六域 DTO 和生成客户端代表性契约测试。
