## 五、关键风险与建议

1. **SQLite → PostgreSQL** 迁移会影响 Prisma schema 和部署方式，建议尽早规划。
2. **ENCRYPTION_KEY** 需要安全分发到所有实例，不能硬编码在 `.env` 中。
3. **邮件集成**依赖外部 IMAP/SMTP 服务器配置，需准备测试邮箱账户联调。
4. **前端代码分割**需注意 Suspense fallback 的加载体验。
5. **测试覆盖**当前覆盖 API 契约和工具函数，业务 E2E 流程（登录→RFQ→报价→审批→订单）是重点缺失项。
