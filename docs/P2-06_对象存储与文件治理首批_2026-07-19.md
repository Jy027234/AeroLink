# P2-06 对象存储与文件治理首批

日期：2026-07-19（Asia/Shanghai）
状态：进行中（本地/S3-compatible 适配器、元数据模型、受控下载和 MinIO 合同验证已落地）

## 已交付

- 新增 `ObjectStorage` 接口及 `LocalObjectStorage`：随机对象键、路径穿越防护、非空文件检查、SHA-256、大小/MIME/版本元数据。
- 新增 Prisma `StoredObject` 元数据模型和 `20260719123000_p2_stored_objects` migration；数据库只保存引用与治理字段，不保存大文件二进制。
- 上传接口在魔数校验后登记元数据，并返回兼容旧 URL 与新的 `/api/files/:id` 受控下载 URL。
- 受控下载按对象 owner/管理角色授权，设置 MIME、大小和安全 Content-Disposition；旧 `/uploads` 入口保留为迁移兼容层，但已改为先查 `StoredObject` 再授权，已登记对象从统一 `ObjectStorage` 流式读取，未迁移历史文件暂仅 admin/manager 可读，不再作为通用认证静态目录。
- 未绑定 owner 的历史对象默认仅允许 admin/manager 读取；普通用户只能读取自己上传且仍为 `AVAILABLE` 的对象，避免通过猜测对象键横向读取历史文件。
- 新增无第三方 SDK 依赖的 `S3CompatibleObjectStorage`（AWS SigV4、Path/Virtual Host 两种寻址、注入式 fetch contract test），通过 `OBJECT_STORAGE_DRIVER=s3` 受控选择；不记录密钥。
- 新增 `server/src/scripts/migrateUploads.ts` 与 `npm --prefix server run storage:migrate-uploads`：默认 dry-run，支持 `--apply`、`--after`、`--limit`、manifest、SHA/大小对账和断点续跑，永不自动删除源文件；manifest 同时记录文件数、总字节和确定性汇总 SHA-256。
- 断点续跑使用同一个 `--manifest` 路径：部分批次会合并已记录的 object key，最终不带 `--after/--limit` 的完整扫描会刷新为 `completeScan=true`；发现 `mismatch` 或 `failed` 时以非零退出，避免把不完整迁移误报为成功。
- 已存在的 object key 只有在 `status=AVAILABLE`、`domain=legacy-upload` 且 SHA/大小同时一致时才记为 `already-migrated`；已删除、不可用或其他业务域的同键对象都会进入 `mismatch`，要求人工处理。
- 新增只读 `server/src/scripts/verifyUploadManifest.ts` 与 `npm --prefix server run storage:verify-manifest`：按 manifest 对源文件、`StoredObject` 元数据和对象存储字节流做三方 SHA-256/大小/状态核验，失败时返回非零退出码，支持显式 `--allow-missing-source` 记录源文件已清理的后置状态。
